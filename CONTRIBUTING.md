# Contributing

## Objetivo
Definir estandares para mantener calidad y consistencia del codigo.

## Requisitos
- Node.js LTS
- npm

## Instalacion
1) `npm install`
2) `npm run dev`

## Estilo de codigo
- Usar ESLint.
- Mantener componentes pequenos y reutilizables.
- Nombres de variables claros y en espanol tecnico.

## Commits
- Usar mensajes tipo: `tipo: descripcion breve`
- Ejemplos: `feat: agrega banner`, `fix: corrige validacion`

## Pull requests
- Describir cambio, riesgo y pasos de prueba.
- Incluir screenshots si hay cambios UI.

## Pruebas
- Ejecutar `npm run lint`.
- Validar flujos principales en navegador.
